Website Accessibility

Home Page Web Content Accessibility Guidelines

How the content of this website is accessible

Making the content of the website accessible means to make the information on the website as understandable as possible, expanding abbreviations and acronyms (like expanding HTML to HyperText Markup Language), as well as refraining from using slang terms, slang acronyms (like idk and lol), and try to use words instead of symbols where it makes sense (like instead of using 5-7, using five to seven).

How this website is coded to be accessible

Coding a website to be more accessible means to use semantic HyperText Markup Language (HTML), allowing interactable elements to be controlled in multiple ways (like mouse and also keyboard controls), and also making sure that your website can be viewed on a range of browsers and devices.

Semantic HyperText Markup Language

Semantic HTML is using the correct HTML elements for their intended purpose as much as possible. An example of this is instead of marking up a button control using a <div> element, using the <button> element makes more sense and requires less work to make the button control work.

If you make websites, you may use non-semantic HTML way more than you realise.

Some examples on non-semantic HTML elements are <div> elements and <span> elements. These elements don't have a specific use, and can be used for anything.

You may be using a <div> or <span> element to separate sections where the <section> or <article> elements would make more sense to use to separate sections, or you may be using a <div> or <span> element to arrange headers and footers where using the <header> and <footer> elements would make more sense.

If you consistently make sure that you use the correct elements for the job as much as possible, you will realise that it makes more sense to use the specific element than using <div> or <span> elements.

Interactable Elements

You can allow interactable elements to be controlled in multiple ways by using the tabindex attribute in HTML or by using JavaScript to add keyboard controls or navigation.

Viewing on other devices

You can also make sure that your website can be viewed on a range of devices and browsers by ensuring that you are using code that is supported by most browsers. To also make sure that your website can be viewed on a range of devices, you can test your website on a range of devices and also test your website on different browsers.

Skip Navigation Links

People percieving your website, using a screen reader or similar technology, may not want to hear navigation links being read aloud. Accessible websites implement a skip navigation link. A skip navigation link (or skipnav) is a link placed before navigation links that allow the user to tab to the skipnav link first (press Tab until you navigate to the skipnav link to see the skipnav link pop out from the top-left of the page) and skip past the navigation to the main content of a website. It is fairly simple to add to a website and can allow people who don't want to hear the links read aloud to skip the links, allowing for a better user experience.
